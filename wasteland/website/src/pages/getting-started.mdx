  <section class="step" id="alias">
    ## Set up a cargo alias <span class="optional">(optional)</span>

    Command line compatible with Cargo. Alias `cargo=abrasive` and keep using cargo the way you normally would.

    Add an alias to your shell config:

    <CodeBlock code={`echo 'alias cargo="abrasive"' >> ~/.bashrc && source ~/.bashrc`} />

    For zsh, replace `~/.bashrc` with `~/.zshrc`. For fish:

    <CodeBlock code={`alias --save cargo="abrasive"`} />

    Now `cargo setup`, `cargo build`, etc. will all route through abrasive.
  </section>

  <section class="step">
    ## Initialize your project

    Navigate to the root of your project and run:

    <CodeBlock code="abrasive setup" />

    Or if you set up the cargo alias:

    <CodeBlock code="cargo setup" />

    This walks you through an interactive setup and creates an `abrasive.toml` in the root of your project.
  </section>
